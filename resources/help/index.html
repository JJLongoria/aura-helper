Aura Helper provides a navigable documentation for help users to use the extension tools.

Commands

Add Apex Comment

Command for create a comment block for methods or apex classes.

Put the mouse on previous method or class line and execute the command.

You can edit the template whenever you want with the command Edit Apex Comment Template

For learn more about templates, click here



Add Javascript Aura Function

Command for create an Aura Function block on JavaScript Aura files.

Put the mouse whatever you want create a function and execute command. Then, you can select the parameters number of a function for create it.



Add Method Block to Doc

Command for create a method block into Aura Documentation File

Block will be created on mouse position into the document. This command provides a list of method for select the method to create.

You can edit the template whenever you want with the command Edit Aura Documentation Template

For learn more about templates, click here



Edit Apex Comment Template

Command for edit the Apex Comment Template used for create the apex code comments automatically.

You can execute this command on any time and it will open an editor window for edit the template

When you finish the template, you can create any comment on apex code with Add Apex Comment command or with code completion tools when type /**

For learn more about templates, click here



Edit Aura Documentation Template

Command for edit the Aura Documentation Template used for create the documentation automatically.

You can execute this command on any time and it will open an editor window for edit the template

When you finish the template, you can create any componente documentation automatically and easy wih Generate Aura Documentation

For learn more about templates, click here



Generate Aura Documentation

Command for create an Aura Documentation File for Aura Components automatically based on user templates and JavaScript standard comment scheme.

                                    
                            /**
                             * functionDescription
                             * @param {paramType} paramName paramDescription
                             */

You can edit the template whenever you want with the command Edit Aura Documentation Template

For learn more about templates, click here



New Aura File

Command for create a new Aura File into the bundle. You can run it in two ways:

  • You can execute the command when Aura Component file (.cmp) is opened on editor
  • You can make a rigth click on Aura Component file (.cmp) in explorar and then select "Aura Helper: New Aura File"

All documents created like in salesforce with the base template except the Aura Documentation File, it will be created based on user template (or default extension template)



Code Completion Tools

Getting Component Attributes

Aura Helper provides code completion tools for Aura Components and JavaScript files for get the Componente Attributes

Only need type v. and press Ctrl+space for get all attribute names. with this provider you can also get the inherited attributes

Work in .cmp and .js files



Getting JS Controller Functions

Aura Helper provides code completion tools for Aura Components files for get JavaScript Controller Functions

Only need type c. and press Ctrl+space for get all function names. with this provider you can also get the inherited functions

Work in .cmp files



Getting JS Helper Functions

Aura Helper provides code completion tools for JavaScript files for get and use JavaScript Helper Functions

Only need type helper. and press Ctrl+space for get all function names. with this provider you can also get the inherited functions

Work in .js files



Getting Apex Methods

Aura Helper provides code completion tools for Aura JavaScript files for get Apex Controller Methods

Only need type c. and press Ctrl+space for get all method names. with this provider you can also get the inherited methods

Work in .js files

Templates

Aura Helper provides to you some tools for create elements based on user defined templates. All the templates used by Aura Helper are in JSON format

You can view the estructure of every template here:

Aura Documentation Template

The template for create Aura Documentation automatically have the next structure: 

                            
                        {
                            "documentbody": [
                                "Line 1 Content",
                                "Line 2 Content",
                                "Line 3 Content",
                                "...",
                                "Line N Content"
                            ],
                            "methodBody": [
                                "Line 1 Content",
                                "Line 2 Content",
                                "Line 3 Content",
                                "...",
                                "Line N Content"
                            ],
                            "paramBody": [
                                "Line 1 Content",
                                "Line 2 Content",
                                "Line 3 Content",
                                "...",
                                "Line N Content"
                            ]
                        }

You can use some keywords for complete the document template.

  • descriptionBody:

    Principal lines of the body. This is the basic skeleton of the Aura Documentation file. You can use in this section the following keywords:

    • {!controllerMethods}
    • {!helperMethods}

    The rest of keywords do not take effect in this section of the template

  • methodBody:

    This lines represent every method body. When Aura Helper generates the documentation file, it copy every method information with this template and then, replace the keywords {!controllerMethods} or {!helperMethods} with the generated body. You can use in this section the following keywords:

    • {!method.description}
    • {!method.signature}
    • {!method.name}
    • {!method.params}

    The rest of keywords do not take effect in this section of the template

  • paramBody:

    Like with methodBody, Aura Helper copy every param information on with this template and the, replace the {!method.params} keyword for the generated body. You can use in this section the following keywords:

    • {!param.description}
    • {!param.name}
    • {!param.type}

    The rest of keywords do not take effect in this section of the template

Example: 

                                
        {
            "documentbody": [
                "<aura:documentation>",
                "\t<aura:description>",
                "\t\t<h6><b>Short description</b> of the component</h6>",
                "\t\t<p>",
                "\t\t\tHelper methods:",
                "\t\t\t<ul>",
                "\t\t\t\t{!helperMethods}",
                "\t\t\t</ul>",
                "\t\t</p>\n",
                "\t</aura:description>",
                "\t<aura:example name=\"ExampleName\" ref=\"ExampleComponent\" label=\"ExampleLabel\">",
                "\t\t",
                "\t</aura:example>",
                "</aura:documentation>"
            ],
            "methodBody": [
                "<li>",
                "\t<b>{!method.signature}: </b> {!method.description}",
                "\t<ul>",
                "\t\t{!method.params}",
                "\t</ul>",
                "</li>"
            ],
            "paramBody": [
                "<li><i>{!param.name} ({!param.type}): </li> {!param.description}"
            ]
        }



Apex Comment Template

The template for create an Apex Comment automatically have the next structure: 

                            
                        {
                            "methodComment": [
                                "commentBody": [
                                    "Line 1 Content",
                                    "Line 2 Content",
                                    "Line 3 Content",
                                    "...",
                                    "Line N Content"
                                ],
                                "paramsBody": "Only One Line",
                                "returnBody": "Only One Line"

                            ],
                            "classComment": [
                                "commentBody": [
                                    "Line 1 Content",
                                    "Line 2 Content",
                                    "Line 3 Content",
                                    "...",
                                    "Line N Content"
                                ]
                            ]
                        }

You can use some keywords for complete the document template.

  • methodComment:

    Section for create the method comment template:

    • commentBody:

      This lines represent a comment method body. You can use in this section the following keywords:

      • {!method.description}
      • {!method.params}
      • {!method.return}

      The rest of keywords do not take effect in this section of the template

    • paramBody:

      This line represent the scheme for each param in the comment. You can use in this section the following keywords:

      • {!param.name}
      • {!param.type}
      • {!param.description}

      The rest of keywords do not take effect in this section of the template

    • returnBody:

      This line represent the scheme for the return statement in the comment. You can use in this section the following keywords:

      • {!return.type}
      • {!return.description}

      The rest of keywords do not take effect in this section of the template

  • classComment:

    Section for create the class comment template:

    • commentBody:

      This lines represent a comment class body. You can use in this section the following keywords:

      • {!class.description}
      • {!class.name}

      The rest of keywords do not take effect in this section of the template

    The rest of keywords do not take effect in this section of the template

Example: 

                            
                {
                    "methodComment": {
                        "paramBody": "## {!param.name} ({!param.type}): {!param.description}",
                        "returnBody": "@@ Return {!return.type}: {!return.description}",
                        "commentBody": [
                            "/**",
                            " * {!method.description}",
                            " *",
                            " * {!method.params}",
                            " * {!method.return}",
                            " */"
                        ]
                    },
                    "classComment": {
                        "commentBody": [
                            "/**",
                            " * {!class.description}",
                            " *",
                            " */"
                        ]
                    }
                }
