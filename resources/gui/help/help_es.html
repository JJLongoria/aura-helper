Resumen de Características

Autocompletado de Código

Aura Helper pone a disposición del usuario múltiples herramientas de autocompletado de código para facilitar la codificación, que pueden ser activadas o desactivadas desde la configuración de la extensión.

Algunas de las funciones son:

  • Capacidad para listar todos los objetos, custom y estándar disponibles en la aplicación. Funciona en las clases Apex
  • Listar los campos de los objetos, incluyendo información como el label, los valores picklist, la longitud... Funciona tanto en clases Apex como en componentes Lightning
  • Mostrar todas las clases de la aplicación, incluyendo las clases de sistema (System, Database...). Funtiona en las clases Apex
  • Obtener y listar las variables y métodos de todas las clases, incluyendo las clases de sistema. Funtiona en las clases Apex
  • Capacidad para crear comentarios de clases y métodos de forma rápida a través de una plantilla definida por el usuario
  • Capacidad para listar la información anterior de forma concatenada (clase.metodo().metodo()...). Funciona tanto en clases Apex como en componentes Lightning
  • Listar todos los componentes lightning custom y estándar de cualquier espacio de nombre (lightning, aura...)
  • Listar los atributos no incluidos en las llamadas a los componentes lightnig, tanto custom como estándar de cualquier espacio de nombre
  • Sugerir los posibles valores de los atributos de los elementos de los componentes lightning, como variants, sizes... etc
  • Capacidad para mostrar las funciones de los controladores Apex, Javascript o el archivo Helper en los archivos de cualquier Bundle de Lightning
  • Soporte para facilitar la construcción de consultas mostrando los campos del objeto indicado en la clausula "from"
  • Más de 100 Snippets de código para facilitar la creación de componentes lightning. Para más detalle ver la sección de Snippets

Interfaz Gráfica de Usuario Avanzada

Otra de las principales características que implementa Aura Helper, es el soporte de una interfaz gráfica avanzada para implementar características más potentes que faciliten el mantenimiento de metadatos o sirvan de soporte para otras tareas.

A través de la interfaz gráfica, Aura Helper provee de una herramienta para crear archivos package.xml de forma rápida, tanto de los metadatos que existen en local, como de todos los metadatos de la organización.

Otra característica implementada a través de la interfaz, es la capacidad para gestionar los ficheros de metadatos como si de Salesforce se tratase. Añade validaciones y comportamiento similares, teniendo en cuenta el contexto y las posibilidades que ofrece VSCode.

Control de Versiones y Metadatos

Para facilitar el trabajo con aplicaciones de control de versiones como Git, Aura Helper contiene varias herramientas para gestionar los archivos de metadatos que simplifican el control de versiones en Salesforce.

Algunas de las herramientas son:

  • Posibilidad de comprimir los archivos XML de forma que ocupen menos líneas y reorganizar el contenido para facilitar su lectura por parte del usuario
  • Interfaz gráfica para gestionar los ficheros de forma más simple y sin salir de VSCode
  • Posibilidad de actualizar los ficheros que se crean en tiempo de ejecucion durante los despliegues en función del contenido, como por ejemplo los perfiles o permission sets, incluyendo toda la información del sistema y sin necesidad de actualizar los demás elementos
  • Capacidad para comprobar qué elementos existen en el entorno y no en local, para poder eliminarlos e igualar el entorno con el contenido local. El usuario puede elegir qué metadatos se borraran
  • Si deseamos realizar despliegues o descargas de metadatos, podemos usar la herramientas de Generador de Paquetes y las opciones de Deploy y Retrieve de SFDX

Documentación

Para facilitar la tediosa tarea de documentación, Aura Helper dispone de herramientas para generar automáticamente o facilitar el trabajo de documentar código

Las principales herramientas son:

  • Capacidad para generar comentarios de clases y métodos Apex siguiendo la estructura de una plantilla definida por el usuario, de forma que se pueda adaptar el estilo de los comentarios al establecido en el proyecto.
  • Soporte para generar de forma automática toda o casi toda la documentación de los componentes Lightning (.auradoc). Usando el esquema de comentarios Javascript, y una plantilla definida por el usuario, se puede generar casi la totalidad de la documentación con un sólo click

    Comandos

    Add Javascript Aura Function

    Este comando permite la creación de un bloque de código que representa una función Javascript en los archivos Aura que permite elegir el número de parámetros que tendrá la función, de esta forma podemos crear funciones más rápido.

    Para utilizar esta característica se debe situar el ratón en la posicion del archivo donde queramos crear la función y ejecutar el comando "Add Javascript Aura Function", elegimos el número de parámetros que queremos para la función y listo.



    Add Method Block to Doc

    Este comando permite añadir un nuevo método al fichero de documentación de Aura siguiendo el esquema establecido en la plantilla. El bloque se creará en la posición del ratón al ejecutar el comando y al ejecutarlo nos proveerá una lista de los métodos disponibles, tanto del helper como del controlador.

    Puede editar la plantilla en cualquier momento con el comando "Edit Aura Documentation Template". Para saber más sobre las plantillas, pulse aquí



    Compress XML File

    Para facilitar la gestión de fichero con programas de control de versiones como GIT, Aura Helper provee un método para comprimir los ficheros XML de los metadatos de forma que sea más fácil de identificar los cambios, combinarlos o desecharlos. También reduce el tamaño de los ficheros permitiendo una gestión más eficiente por parte de Git o similares.

    Para ejecutar este comando tenemos varias opciones.

    • Ejecutar el comando con un fichero XML abierto para comprimir dicho archivo
    • Hacer click con el botón derecho en el fichero abierto y ejecutar el comando desde el menú contextual
    • Hacer click derecho en el fichero en el explorador de archivos y ejecutar el comando desde el menú contextual



    Compress All XML Files

    Al igual que podemos comprimir un sólo fichero, podemos comprimir todos los ficheros de metadatos del proyecto o de una carpeta concreta.

    Para ejecutar este comando tenemos varias opciones.

    • Si ejecutamos el comando desde la paleta de comandos, se comprimiran todos los ficheros
    • Hacer click derecho en la carpeta raíz del proyecto y ejecutar el comando desde el menú contextual. Esto comprimirá todos los ficheros
    • Hacer click derecho en una carpeta concreta y ejecutar el comando desde el menú contextual. Esto comprimirá todos los ficheros de esa carpeta, incluyendo las subcarpetas



    Edit Apex Comment Template

    El comando "Edit Apex Comment Template" permite editar la plantilla que se usará para crear los comentarios de las clases y métodos Apex. Esta plantilla puede ser editada por el usuario siguiendo las reglas indicadas en las seccion de plantillas para así adaptar la documentación del proyecto a los estándares establecidos por el usuario

    Para saber más sobre las plantillas, pulse aquí



    Edit Aura Documentation Template

    Este comando, similar al comando para editar la plantilla de comentario de Apex, permite crear y editar una plantilla para crear la documentación de los componentes Lightning de forma automática.

    Para saber más sobre las plantillas, pulse aquí



    Generate Aura Documentation

    Este comando permite crear la documentación Aura de los componentes lightning de forma automática basada en la plantilla definida por el usuario y el esquema básico de comentarios de Javascript 

                                
                        /**
                         * functionDescription
                         * @param {paramType} paramName paramDescription
                         */

    Puede editar la plantilla de la documentación cuando desee ejecutando el comando "Edit Aura Documentation Template"

    Para saber más sobre las plantillas, pulse aquí



    Match Org With Local

    El comando "Match Org With Local" compara todos los metadatos que existen en el proyecto local con los metadatos que existen en la aplicación para, posteriormente, ofrecer al usuario una vista con los metadatos que existen en la organización pero que no existen en local. El proceso puede tardar varios minutos dependiendo de la conexión a Internet y las prestaciones del equipo.

    En la vista mostrada, el usuario puede elegir qué metadatos quiere borrar de la organización para igualar el entorno con el proyecto local y, al hacer click en "Delete", se eliminarán de la organización los metadatos seleccionados. Este proceso es bastante más rápido que la comparación de datos, a excepción de producción, donde se ejecutarán las clases de test, lo que puede alargar el proceso.



    New Aura File

    Con este comando podemos crear cualquier fichero dentro del paquete Aura. Podemos ejecutarlo de dos formas:

    • Se puede ejecutar este comando cuando un archivo de Componente Aura (.cmp) está abierto en el editor
    • se puede hacer click derecho en el archivo de Componente Aura (.cmp) o en la carpeta del paquete del componente y seleccionar el comando en el menú contextual
    Tras la ejecución de este comando se nos mostrará una lista para elegir el tipo de fichero que deseamos crear. Sólo apareceran los ficheros que no existan en el Bundle.

    Todos los ficheros se crearán con la estructura por defecto, excepto el fichero de documentación, que se creará con la plantilla definida por el usuario (o la plantilla por defecto)



    Refresh Metadata Index

    Para ofrecer una mejor experiencia de autocompletado de código, Aura Helper tiene comandos para refrescar el índice de objetos y sus campos. Para actualizar este índice, debemos ejecutar este comando

    Esta operación puede tardar varios minutos dependiendo del número de objetos de la organización, pero sólo es necesario ejecutarlo por completo una vez, ya que Aura Helper dispone de otro comando para actualizar el Indice de un Objeto, siendo una ejecución mucho más rápida.



    Refresh Metadata Index for Object

    Al igual que podemos actualizar el índice para todos los objetos de la organización, podemos hacerlo para un sólo objeto. Esta operación es significativamente más rápida que la actualización completa del índice.

    Esta operación puede tardar varios minutos, dependiendo de la velocidad de conexión y el número de objetos de la organización.

    Para actualizar el índice completo ejecute el comando "Refresh Metadata Index"



    Retrieve Full Permission Set(s)

    Este comando permite actualizar los archivos de permissions sets almacenados en local con la totalidad de los permisos, al contrario que un retrieve normal, que se trae información básica si incluimos otros metadatos como classes, campos, objetos...

    Sólo afecta a los ficheros de permisos y permite la actualización de uno o todos los permisos, tanto los almacenados en local como todos los de la organización.



    Retrieve Full Profile(s)

    Este comando permite actualizar los archivos de perfiles almacenados en local con la totalidad de los permisos, al contrario que un retrieve normal, que se trae información básica si incluimos otros metadatos como classes, campos, objetos...

    Sólo afecta a los ficheros de perfiles y permite la actualización de uno o todos los perfiles, tanto los almacenados en local como todos los de la organización.

    Herramientas de Autocompletado

    Aura Helper provee múltiples herramientas de autocompletado de código tanto para Apex como para Lightning, permitiendo una programación más fácil y eficiente.
    Al mismo tiempo dispone de múltiples snippets que pueden verse más detenidamente en la sección de Snippets

    Obtener, llamar y crear Componentes

    Una de las herramientas de autocompletado de código de las que dispone Aura Helper es la posibilidad de obtener, llamar y contruir componetnes, eventos y aplicaciones Lightning

    Formas de uso:

    • Escribir c: y presionar Ctrl+space para obtener una lista de todos los componentes, eventos y aplicaciones del espacio de nombre "c"
    • Si se escribe <c:ComponentName y se presiona Ctrl+space se obtendrá una lista de todos los atributos del componente, apareciendo únicamente los que no están ya añadidos al componente
    • Una vez escrito el atributo, Aura Helper puede sugerirnos los posibles valores admitidos presionando Ctrl+space
    • Para llamar componentes Standard (aura, lightning, ui...), únicamente debemos escribir <Namespace:ComponentName y presionar Ctrl+space para obtener una lista de los atributos del componente
    • También se disponen de múltiples snippets para crear todo tipo de componentes y sus cariantes, revisar la seccion de Snippets

    Funciona tanto en archivos .cmp como .js y puede activarse y desactivarse desde la configuración de la extensión



    Obtener Attributos de Componentes

    Del mismo modo que se pueden obtener los atributos de los componentes para crearlos de forma rápida, se pueden obtener estos para referenciarlos en el componente. Para hacer esto, úncamente se debe escribir v. y presionar Ctrl+space para obtener todos los atributos. También se obtendrán los atributos heredaros.

    Si el atributo es algún SObject, se podrá obtener una lista de sus campos, incluso concatenarlos con otros campos en caso de relaciones. v.sObjectType.field.field.field.... Para sacar todo el potencial de esta función, es necesario actualizar el Indice de Objetos

    Funciona tanto en archivos .cmp como .js y puede activarse y desactivarse desde la configuración de la extensión



    Obtener Functiones del Controlador JS

    Aura Helper también provee otras herramientas de autocompletado de código que permite obtener los métodos del controlador JavaScript desde los componentes

    Para hacer uso de esta característica, úncamente se debe escribir c. y presionar Ctrl+space para obtener los nombres de las funciones. Esto también obtiene las funciones heredadas

    Funciona tanto en archivos .cmp como .js y puede activarse y desactivarse desde la configuración de la extensión



    Obtener Functiones del Helper JS

    Al igual que con los métodos del controlador, Aura Helper puede obtener los métodos del helper para facilitar la codificación

    Para hacer uso de esta característica, úncamente se debe escribir helper. y presionar Ctrl+space para obtener los nombres de las funciones. Esto también obtiene las funciones heredadas. Para facilitar el uso, se recomienda identificar el parámetro del helper con el nombre "helper"

    Funciona en archivos .js y puede activarse y desactivarse desde la configuración de la extensión



    Obtener Métodos Apex en JS

    Del mismo modo que con los métodos del controlador o el helper, Aura Helper también dispone de herramientas para obtener los métodos del controlador Apex

    Para ello, únicamente se debe escribir c. y presionar Ctrl+space para obtener el nombre de los métodos. Como en el resto de casos, también obtiene las funciones heredadas

    Funciona en archivos .js y puede activarse y desactivarse desde la configuración de la extensión



    Soporte en Consultas

    Aura Helper contiene también herramientas para ayudar a completar las consultas permitiendo listar los campos del objeto a consultar

    Una vez escrita la estructura mímina de una consulta Select Id from [SObject], podemos obtener los campos indicados en el objeto tras el from presionando Ctrl+space en la clausula de proyección. Esta característica funciona tanto para las queries tanto en componentes lightning, ficheros Javascript o clases Apex, tanto para queries entre [] como queries entre ''

    Para aprovechar al máximo esta característica, es necesario actualizar el Indice de Objetos

    Funciona tanto en archivos .cmp como .js y .cls y puede activarse y desactivarse desde la configuración de la extensión



    Soporte para Código Apex

    Por último, como no podía ser menos, Aura Helper también dispone de múltiples herramientas para autocompletar código Apex

    Desde listar todos los objetos disponibles, Estandar o Custom, como las clases creadas en la aplicación, sus campos, métodos, etc.
    También tiene soporte para concatenar llamadas de distintos tipos de datos o métodos como className.methodName().fieldName. Todas las operaciones se hacen con la información disponible en el proyecto local. Si se quiere trabajar con los últimos cambios será necesario refrescar el proyecto.

    Otra característica disponible es la posibilidad de crear comentarios de clases y métodos basados en una plantilla escribiendo /**

    Para aprovechar al máximo esta característica, es necesario actualizar el Indice de Objetos

    Funciona en archivos .cls, puede activarse y desactivarse desde la configuración de la extensión y tiene soporte para clases y objetos custom y estándar



    Documentación (Plantillas)

    Aura Helper dispone de herramientas para crear diferentes elementos basados en plantillas definidas por el usuario. Todas las plantillas usadas por Aura Helper están formato JSON
    A continuación se muestra la estructura de cada plantilla

    Plantilla de Documentacion Aura

    Para facilitar la creación de la documentación de componentes Aura, se puede definir una plantilla de forma que se genere la mayoría del código de la documentación de forma automática. Lo que permite crear la documentación de forma más rápida y sencilla.

    La plantilla para crear la Documetnación de Aura de forma automática tiene la siguiente estructura:

                                
                        {
                            "documentbody": [
                                "Line 1 Content",
                                "Line 2 Content",
                                "Line 3 Content",
                                "...",
                                "Line N Content"
                            ],
                            "methodBody": [
                                "Line 1 Content",
                                "Line 2 Content",
                                "Line 3 Content",
                                "...",
                                "Line N Content"
                            ],
                            "paramBody": [
                                "Line 1 Content",
                                "Line 2 Content",
                                "Line 3 Content",
                                "...",
                                "Line N Content"
                            ]
                        }

    Se pueden usar distintas Palabras Clave para completar la plantilla de documentación.

    • descriptionBody:

      Lineas principales del cuerpo. Es el esqueleto básico para el fichero de Documentación de Aura. En esta sección se pueden usar las siguientes Palabras Clave:

      • {!controllerMethods}
      • {!helperMethods}

      El Resto de Palabras Clave no tendrán efecto en esta sección de la plantilla

    • methodBody:

      Estas lineas representan la estructura de cada método en la documentación. Cuando Aura Helper genera el fichero de documentación, copia la información de cada método con la estructura definida en esta plantilla y, después, reemplaza las Palabras Clave {!controllerMethods} o {!helperMethods} con el contenido generado. En esta sección se pueden usar las siguientes Palabras Clave:

      • {!method.description}
      • {!method.auraSignature}
      • {!method.signature}
      • {!method.name}
      • {!method.params}

      El Resto de Palabras Clave no tendrán efecto en esta sección de la plantilla

    • paramBody:

      Estas líneas, por último, representan la estructura de cada parámetro de los métodos. Como en la sección "methodBody", Aura Helper copia la información de cada parámetro siguiendo esta plantilla, y reemplaza la Palabra Clave {!method.params} con el contenido generado. En esta sección se pueden usar las siguientes Palabras Clave:

      • {!param.description}
      • {!param.name}
      • {!param.type}

      El Resto de Palabras Clave no tendrán efecto en esta sección de la plantilla

    Ejemplo: 

                                    
        {
            "documentbody": [
                "<aura:documentation>",
                "\t<aura:description>",
                "\t\t<h6><b>Short description</b> of the component</h6>",
                "\t\t<p>",
                "\t\t\tHelper methods:",
                "\t\t\t<ul>",
                "\t\t\t\t{!helperMethods}",
                "\t\t\t</ul>",
                "\t\t</p>\n",
                "\t</aura:description>",
                "\t<aura:example name=\"ExampleName\" ref=\"ExampleComponent\" label=\"ExampleLabel\">",
                "\t\t",
                "\t</aura:example>",
                "</aura:documentation>"
            ],
            "methodBody": [
                "<li>",
                "\t<b>{!method.signature}: </b> {!method.description}",
                "\t<ul>",
                "\t\t{!method.params}",
                "\t</ul>",
                "</li>"
            ],
            "paramBody": [
                "<li><i>{!param.name} ({!param.type}): </li> {!param.description}"
            ]
        }



    Plantilla de Comentarios Apex

    Para facilitar la documentación de las clases Apex, Aura Helper también dispone de una plantilla que se puede definir por el usuario para establecer un esquema de comentarios, tanto para los comentarios de clases como de los métodos.

    La plantilla para crear Comentarios Apex de forma automática tiene la siguiente estructura: 

                            
                    {
                        "methodComment": [
                            "commentBody": [
                                "Line 1 Content",
                                "Line 2 Content",
                                "Line 3 Content",
                                "...",
                                "Line N Content"
                            ],
                            "paramsBody": "Only One Line",
                            "returnBody": "Only One Line"

                        ],
                        "classComment": [
                            "commentBody": [
                                "Line 1 Content",
                                "Line 2 Content",
                                "Line 3 Content",
                                "...",
                                "Line N Content"
                            ]
                        ]
                    }

    Se pueden usar distintas Palabras Clave para completar la plantilla de documentación.

    • methodComment:

      Sección para crear la estructura de los comentarios de los métodos Apex:

      • commentBody:

        Líneas principales del comentario. Representa el esqueleto básico de los comentarios Apex para los métodos. En esta sección se pueden usar las siguientes Palabras Clave:

        • {!method.description}
        • {!method.params}
        • {!method.return}

        El Resto de Palabras Clave no tendrán efecto en esta sección de la plantilla

      • paramBody:

        Estas líneas representan el esquema básico para cada parámetro de los métodos en el comentario. Al generar el comentario, Aura Helper copia la información de cada parámetro de los métodos siguiendo la estructura definida y y sustituye la Palabra Clave {!method.params} con el contenido generado. En esta sección se pueden usar las siguientes Palabras Clave:

        • {!param.name}
        • {!param.type}
        • {!param.description}

        El Resto de Palabras Clave no tendrán efecto en esta sección de la plantilla

      • returnBody:

        Las líneas definidas en esta sección representan la estructura básica de la información devuelta por el método. Como con los parámetros, Aura Helper copia la información correspondiente usando la estructura indicada y reemplaza la Palabra Clave {!method.return} por el contenido generado. En esta sección se pueden usar las siguientes Palabras Clave:

        • {!return.type}
        • {!return.description}

        El Resto de Palabras Clave no tendrán efecto en esta sección de la plantilla

    • classComment:

      Sección para crear la estructura de los comentarios de las clases Apex:

      • commentBody:

        Líneas principales del comentario. Representa el esqueleto básico de los comentarios Apex para los métodos. En esta sección se pueden usar las siguientes Palabras Clave:

        • {!class.description}
        • {!class.name}

        El Resto de Palabras Clave no tendrán efecto en esta sección de la plantilla

    Example: 

                            
            {
                "methodComment": {
                    "paramBody": "## {!param.name} ({!param.type}): {!param.description}",
                    "returnBody": "@@ Return {!return.type}: {!return.description}",
                    "commentBody": [
                        "/**",
                        " * {!method.description}",
                        " *",
                        " * {!method.params}",
                        " * {!method.return}",
                        " */"
                    ]
